## Problem Statement

A complete beginner (zero coding experience) wants to:
1. Learn AI engineering from scratch and land an entry-level AI/LLM application developer job within 6 months
2. Grow an Instagram account from under 500 followers to 10,000+ using their AI learning journey as the content niche
3. Track both tracks publicly via a live GitHub Pages dashboard with visual progress rings/streaks

The two goals are not competing — they compound. Learning AI creates authentic content. Content builds brand with AI employers. The dashboard itself is the first real portfolio project.

**Target role:** Junior AI/LLM application developer at an AI-native startup (found via Wellfound/AngelList). Prerequisites: 3 deployed projects with API integrations, public GitHub portfolio, strong README documentation.

## What Makes This Cool

You don't want to learn. You want to be *seen* learning. The public dashboard is accountability architecture — quitting becomes public. It's also an open source project that other beginners can fork, which means by Month 6 you have GitHub stars, forks, and a product people actually use. That's stronger than any certificate.

Each week you ship one new dashboard feature. That feature is both a coding practice session AND a Reel. The content makes itself.

## Constraints

- Zero coding background — learning order matters: Python → Git/HTML → Dashboard → ML → LLMs
- 3+ hours/day available, but split as: 2h mornings = code, 1h evenings = content — never negotiated day-to-day
- Under 500 Instagram followers starting point
- Context switching across 4 modes (code/content/dashboard/audience) is the burnout risk — hard time-blocking is not optional
- 10K Instagram in 6 months requires one high-share moment in Month 6 (the "I got the job" outcome) — the trajectory builds toward it, but the final sprint depends on a viral anchor Reel

## Premises

1. Learning content appears from Week 1 (journey meta-content: "I'm starting from zero"). Deep technical AI content starts Month 3. This is not a contradiction — the journey IS the content.
2. 3+ hours/day with hard time-blocking enables the compressed 6-month timeline. Without time-blocking, the schedule collapses.
3. Under 500 followers → 10K in 6 months assumes: Reels-first (3x/week in Months 1–2, scaling to 5x/week in Months 3–6), 2 collab posts per month from Month 2, and a single viral anchor Reel in Month 6 (100K+ views scenario)
4. The two tracks ARE the same activity from two angles: learning AI → creates content → builds brand with employers
5. The GitHub Pages dashboard IS the first real portfolio project — it counts as one of your 3 required portfolio items

## Approaches Considered

### Approach A: The Tracker
Simple markdown weekly logs + flat HTML GitHub Pages dashboard. Week files fill in; a JSON file drives the visual rings. Ships in 2 days, manual updates, stays personal.

### Approach B: AI Learning OS (selected)
Full public open source system. GitHub Pages with progress rings reading from progress.json, weekly GitHub Actions CI/CD, skill tree by month, project gallery. Dashboard is the Month 1–2 portfolio project. New feature each week = Reel content. Forkable by other learners.

### Approach C: Reels-First Learning
Every concept becomes a 60-second Reel before moving to the next topic. Dashboard tracks Reel count + topics covered. Requires camera comfort every session.

## Recommended Approach: AI Learning OS

Turns the first coding project into a portfolio artifact AND a weekly content engine simultaneously. The "forkable" design means other learners use it, which creates GitHub social proof visible to employers. Each week's new dashboard feature is both a coding practice session and Reel content ("I just added a skill tree to my AI Learning OS").

## The 6-Month Roadmap

**Daily schedule (non-negotiable):**
- 9am–11am (or any 2h morning block) = code only, no Instagram
- 7pm–8pm (or any 1h evening block) = content creation/editing only, no new code
- Weekends: batch-produce next week's static posts + any catch-up coding

**Recovery Protocol (read this before Month 1):**
If you miss more than 3 days in a row: skip the dashboard feature for that week, post one Reel instead of the full week's count, and continue coding from where you left off. The streak resets. The plan does not. Do not try to catch up on missed posts — move forward.

### Month 4 — LLMs + APIs + Dashboard v4 ("The AI Features")

**Dashboard v4 features:**
